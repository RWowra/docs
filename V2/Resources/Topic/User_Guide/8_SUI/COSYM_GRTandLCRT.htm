COSYM_GRT Target in MATLAB®/Simulink®

COSYM allows import of simulink models for which the code was generated with the COSYM_GRT target in MATLAB®/Simulink®. The behavior of a simulink model will be unaffected from the target selected for code generation. The difference between the two targets defines the interface of inports/outports of your simulink model that will be visible in COSYM as well as in the number of measurement variables that are generated automatically. The following descriptions aim at giving an overview of both approaches to yield a decision help which target might fit better.

Using the COSYM_GRT target for code generation, all inports/outports that are available on the top-level simulink model hierarchy will be available in COSYM for connecting them with other models. Measurement variables will be generated for all outports of subsystems on any model hierarchy level. If you want to inspect additional values, you can set test points explicitly in Simulink®.

In COSYM, you can specify the order of the model execution in the Deployment view by mapping computational processes of models in tasks. In Simulink, each simulation cycle distinguishes several phases. During such a cycle, two important phases are the calculation of model outputs as well as the numeric integration when updating the states.
