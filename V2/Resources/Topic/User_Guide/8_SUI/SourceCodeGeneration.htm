Source Code Generation

Source code for the model can be generated in MATLAB® using COSYM Real Time (COSYM_GRT) target.

COSYM manages to set the path of the MATLAB scripts to generate source code for the model, so that it is readily available for you.

Note 

Target has to be associated with MATLAB® for Code generation. You can select the MATLAB version(s) while installing COSYM.

If MATLAB® application is installed after COSYM installation, then you have to associate the target manually.

To associate the target manually, you have to run the MATLABassociation.exe file in admin mode which is available at <COSYM installation directory>\MATLAB folder. "Associate with MatLab" dialog box is displayed with the list of all installed MATLAB versions. Select the required MATLAB version(s) to which the target has to be associated. Click OK to apply the changes.

Refer to "Associate with Matlab" section in the COSYM_V3.4.1_Getting_Started_Guide.pdf document for more details.

Before generating the source code, follow the steps mentioned in the sections linked below to setup the model configuration.

  • Configuring the Settings for Source Code Generation of COSYM_GRT Target

Once the settings are done, you can generate the source code for a model. On successful code generation, <ModelName>*.slxip is generated in the present working directory.

Note 
  • You should not change the name of the *.slxip file which is generated.
  • Consecutively generating the source code for the same model will overwrite the existing generated code.
  • Models with LCO port(s) is(are) not supported in the COSYM_GRT target. In case, a model has LCO port(s) and COSYM_GRT is chosen as the target, then the error message "COSYM_GRT: LCO Ports found in model, cosym_grt does not support LCInport/LCOutport blocks" is thrown and code generation is stopped.
