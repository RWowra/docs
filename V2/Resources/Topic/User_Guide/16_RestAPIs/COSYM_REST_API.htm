COSYM REST APIs

The section helps to understand the REST API Interface exposed by COSYM. COSYM uses the Swagger framework. With this framework, the server, client, and documentation are synchronized.

Swagger is a specification for documenting REST APIs, where clients can easily understand without any prior knowledge of server implementation or access to the server code.

COSYM REST API responses are in JSON format. COSYM REST API clients can currently be generated for Java, JavaScript, C#, and Python language only.

The document also explains about the demo on REST APIs which is shipped with the COSYM Product. Refer to Simple SiL Simulation Automated with REST APIs.

Note 
    COSYM REST API operates on resources which are mainly hierarchical in nature. It is recommended to load resources at top level before operating on resources which are at the next level in hierarchy. This is also done to improve the performance as COSYM persists its metadata in a file system. Hence, always load (read) project before performing any operation on project resources. Similarly, load (read) the system before calling build API on system. This needs to be followed in the Swagger Interface as well.
    It is recommended not to use the COSYM application via GUI while running scripts using the COSYM REST APIs. Therefore, close the currently opened project in GUI before working on scripts using COSYM REST APIs.
