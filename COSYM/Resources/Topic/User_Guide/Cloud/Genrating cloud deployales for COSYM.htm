Generating Cloud Deployables for COSYM

The simulation for a system can be executed either in ETAS EE or on cloud.

For simulation on cloud, it is necessary to make the settings on the Settings menu before building a system.

Make sure that you have enabled the "Create cloud deployable" in the "Product Settings". Refer to Making Settings from the Project Screen for more details.

You can specify the core value and the required memory in the system.config file to execute the simulation process on cloud.

To update the system.config file

  • Go to <COSYM installation directory>COSYM_KARAF\cosym\appdata\ATS_CodeGen_Resources folder.
  • Open the system.config file in any text editor.
  • Specify the value for noOfCoresForThisNode and minimumRequiredMemory. The default values are shown in the Fig. 195.

Fig. 195: system.config file

Once the build is done for a system, the cloud deployables are generated and it is available at <COSYM project location>\Systems\SystemATS\Configuration\OS folder. A dialog box is displayed (Fig. 189) once the build is done which also displays the path of the generated cloud deployables.

These files will be used for simulation on cloud.
