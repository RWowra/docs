Importing a Model Instance

The "Import Model Instance" feature imports a model instance in a *.csmi format.

To import a model instance

  • Right-click on a system.
  • A context menu is displayed.
  • Click .
  • The "Import Model Instance" dialog box is displayed.

Fig. 155: The "Import Model Instance" dialog box

  • Specify the location of the source model instance to import it or click Browse to select it.
  • Specify the location to save the model of that instance in the library or click Browse to select it. By default, it will be saved in the "Control" folder if destination is not provided.
    It is applicable only for the library models. The hardware models will be imported into a target which is mapped to that system.
  • Check the checkbox, "Overwrite existing connections" to overwrite the existing connections.
  • Click Import.
  • The *.csmi format of a model instance is imported into a system.

If the selected source model instance is of older version (Supporting Versions of COSYM for Migration), then COSYM migrates it into the current version internally and replace it.
