ASCMO-STATIC Main Window

The main working window of ASCMO-STATIC shows the modeled dependencies of the outputs on the inputs as intersection plots and is also called ISP view. The following figure shows the main user interface.

Fig. 16: Graphical user interface (GUI) of ASCMO-STATIC (with opened project)

The GUI is divided into 5 different parts:

  • ➀ Main menu
  • ➁Toolbar
  • ➂Outputs
  • ➃ Main working window: Intersection plots (ISP View) and Inputs
  • ➄Log window
  • Status bar (footer) with current state information

Intersection Plots

The model training for an output leads to a function of the form

youtput = f(xinput_1,...,xinput_n)

With respect to the display, this results in a hyperplane in a n+1-dimensional space that can graphically no longer be displayed as soon as n > 2.

Instead, the ASCMO-STATIC main window shows n intersections through this hyperplane, the so-called intersection plots. With each of these intersections, only one dimension is being varied, while the values of the other dimensions are kept constant, resulting in

youtput = f(xinput_1),..., youtput = f(xinput_n)

With n inputs and m outputs, this results in n x m intersection plots – the inputs x1,..., xn (in the user interface underneath the intersection plots) and the outputs y1,..., ym (to the left of them) form a matrix at whose intersections the respective intersection plots y1 = f(x1),..., ym= f(xn) are displayed.

The context menu of the intersection plots

Right-clicking an intersection plot opens a context menu with the following functions:

  • Set Current Model as Reference for '<output>'*

    Sets the current model of the output <output> as reference.
    This makes it possible to display the influence of a change of modeling parameters (hyper-parameters, outliers, Box-Cox transformation etc.).

  • Show Reference Model for '<output>'*

    Shows the plot of the reference model in blue beside the current plot of the relevant output (black).

  • Set All Models as Reference*

    Shows the plot of the reference model for all models in blue beside the current plots (black).

  • Manual Output Range '<output>'

    Manual input of minimum and maximum of the y-axis.

  • Autoscaled Output Range '<output>'

    Scales the y-axis according to the minimum and maximum of the output.

  • Manual Input Range '<input>'

    Manual input of minimum and maximum of the x-axis.

  • Autoscale Input Range to Min/Max of '<input>'

    Scales the x-axis according to the minimum and maximum of the input.

  • Autoscale Input Range to Hull on '<input>'

    Scales the x-axis according to the minimum and maximum of the selected input data hull; see also Configure Convex Hull on Inputs and Show Hull On Inputs (green).

  • Autoscale Input Range to Bounds on '<input>'

    Scales the x-axis according to the bounds of the input data at the operating point currently set (see also Show Inputs Bounds/Map Bounds over OP (yellow)).

  • Set Discrete Values for Input '<input>'

    If a model input can only accept discrete values, they can be defined here.

  • Set Sigma for Uncertain Input '<input>'

    In the case of a noisy input, it can happen that the model of an output returns a minimum/maximum that cannot be set at the input with the required precision - selecting a corresponding value for sigma avoids this.

Note  

* This item is available when the Advanced Settings are enabled.

Inputs

The inputs x1,..., xn are displayed under the intersection plots – they represent the respective X axis of the intersection plots (in the figure, these are "NOx_rel = f(injection)" and "NOx_rel = f(ignition)").

  • A Input name

    Click the button to change the name or unit and parameters like setting the visible range or configuring the input in the "<input> - Parameters" window.

    Additional information are shown by hovering the button.

  • B Current value of the input

    The value can be edited directly in the input field or by clicking in the intersection plot. The unit is shown on the right of the value.

  • C Hide this input

    Click to remove the input from the display (the same as deactivating the Show option in the "Input Properties" window).

    The status (locked/unlocked/use map) is not changed when you remove an input from the display.

  • D Status

    Click to change the state of the current input. The following states are available:

    unlocked

    The value can be changed manually and by the optimization.

    The input name appears in blue, and the value field (2) can be edited.

    locked

    The value is fixed. It cannot be changed manually or by the optimization.

    The input name appears in gray, and the value field (2) is set to read-only.

    use map

    The input value is interpolated from the respective calibration map at the point of the current speed and load.

    The input name appears in red, and the value field (2) is set to read-only.

    The status (locked/unlocked/use map) is kept when you hide an input.

  • D Hidden inputs

    All inputs not displayed in the ISP view are listed here. Double-click on an entry to show it in the ISP view.

Outputs

The modeled outputs are displayed to the left of the intersection plots.

  • A Output name

    Click the button to change the model type, name, unit, and the visible range in the "Output Properties for <output>" window.

  • B Model type of the output

    Click on the model type name to change the model parameters in the the "<output> - Parameters" window.

  • C Current value of the output

    The unit is shown on the right of the value.

Log Window

In this area, notes, status messages and results (e.g. of optimization runs) are shown.
