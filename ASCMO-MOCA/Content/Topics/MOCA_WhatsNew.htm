What's New in ASCMO-MOCA V5.14

General

  • Data Import has an option to split the data into multiple records on NaN (not a number). This works for both MDF and XLSX files. Note that empty rows in XLSX are not taken into account, so use a line of text for the split.

    ASCMO Data Import (ASCMO-DYNAMIC)

    Additional Data to ASCMO Project

ASCMO-MOCA

  • In the Visualization step, you can now export and import tab configurations. The exported file can be manually modified, e.g. to duplicate a tab with modified signals, manually modify the exported file and import it as a new tab.

    Visualization Step

  • The list of parameter sets during optimization now includes the running number of the multistart, e.g., Multistart 1, Iteration 1.

  • An appropriate line style is automatically selected when a signal plot is created, e.g. a line is selected if the x-axis is time.

    Element Configuration

  • The visualization remembers the zoom setting when it is changed manually. Automatic zoom (Home button) now takes cursors and selections into account.

  • The Measured vs. Predicted plot allows you to define a limit, which is displayed as two lines and shows the number of points outside the limit.

    Element Configuration

  • Visualizations can now be exported using the exportVisualizationToImage interface method.

  • New Parameters menu that collects actions on parameters such as export and duplicate.

    Parameters Menu

  • MOCA now stores parameter sets more efficiently in the saved project file and keeps all temporary parametersets. The new optimizer option Save every nth result (Optimization step > Configure) can be used to control the number of saved temporary parametersets, e.g. save every second parameterset.

    Optimizer Options

  • MOCA now allows training of data based models (machine learning, black box models like ASCMO Static/Dynamic). The new step "Data Based Modeling" requires an ASCMO Dynamic license. Closed optimization of data-based and physics-based models is not possible, but optimization can be done iteratively, e.g. first optimize the MOCA function that feeds data into the data-based model and then train the data-based model.

    Data Based Modelling Step

  • The table in the Data step and the table in the Visualization step now have a comment column where you can enter text. For example, use the selection in multiple scatter plots to find the intersection points and comment on those points.

    Data Step

    Element Configuration

  • Data in visualizations can be filtered. There is one global filter that works on all MOCA steps and local filters that can be applied to specific visualization plots.

    Filter Data

    Edit Filter

    Filtering Data

    Element Configuration

  • Parameter plots now show all data points instead of 10,000.

  • "dT" is now available as a dependent parameter variable.

  • Support for automatic group axis mapping of text parameters from an FMU.

  • Functions that use matrix operations (e.g., matrix multiplication, transpose, index to matrix) can now be exported to FMU and used for optimization with code generation, which provides faster optimization when using timedelay.

  • New Plot Type: Parameter Contour Plot

    Element Configuration

    Visualization

  • You can now create a report from a PowerPoint template.

    Visualization Step

    Report Generator
