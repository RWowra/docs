Tab. 1: Available export formats for parameter types (p: only parameter export available, pb: export of parameters and/or bounds available)

To export parameters and/or bounds, proceed as follows.

  1. Do one of the following:

    • Open the Parameters pane and go to the "Parameters" tab.

    • Open the Optimization pane and go to the "Optimize" tab.

    Note  

    An export from the Parameters pane exports the reference parameters, an export from the Optimization pane exports the optimization parameters.

  2. In that pane, click on the button next to the Export button and select the export method Export Parameters, Export Bounds, or Export Parameters and Bounds from the drop-down menu.

    Tab. 1 lists the export methods available for each combination of export format and parameter type. Items that cannot be exported (e.g., bounds for a cube parameter) are skipped, and warnings are issued in the log window.

    A file selection window opens.

  3. In the file selection window, enter or select path, name, and format for the export file.

    Note  

    Bounds cannot be exported to *.m files.

  4. Click on Save.

    If you selected a DCM, CSV, Excel or Calibration data file format, the parameters and/or bounds are exported. Impossible combinations of parameter type, export format, and export method are ignored.

    If you selected the M-script format, continue as follows.

    The "Export to M-script" window opens. The table is the same as the mapping table (Mapping table (Parameters)) in the "Parameters Mapping" area in the lower part (Simulink variant) of the Models pane.

  5. In the "Export to M-script" window, do the following:

    1. Click on Create Default to map all parameters in your project to default Simulink parameters.

    2. Click on Get from Simulink to map the MOCA parameters to parameters with identical names in the Simulink model.

      Note  

      Only possible if your project contains a Simulink model.

    3. Map the MOCA parameters manually.

      column name

      action

      "MOCA Parameter"

      Select a parameter from the combo box. 

      Note  

      Compressed model parameters cannot be mapped.

      "Table Data Simulink" 

      Enter variable names to be used in the block mask.

      "Breakpoints <n> Simulink" 

    4. If necessary, edit the values in the other columns.

    5. If the desired mappings are defined, click on Export.

    The parameters are exported. Incomplete mappings are ignored. Each compressed model parameter in your project is exported to a separate file named <filename>_<ComprModel_parametername>.m.
