Calibration Variables (FMU, COSYM-GRT target based MATLAB®/Simulink® and LCO C-Code Models)

The "Calibration Variables" view helps to add the calibration variables into a model. This is applicable for all types of models such as FMUs, COSYM-GRT target based MATLAB®/Simulink® models, LCO C-Code models (except COSYM C-Code models).

The GUI and the operations in this view differ when compared to those in C-Code model. For example, the "Type" column contains the data type and structure type information.

Add Parameter

To add a calibration variable

  • Click .
  • The "Add Parameter" window is opened.
  • The available parameters are automatically loaded.
  • All the fields in the "Add Parameter" window are non-editable except the "Structure type" field.
  • The name will always be taken from the model field.
  • The "Data orientation" shows the internal identified information such as "Column/Row" of the model.
  • The "Variable path" is displayed based on the internally used path variable (mainly "Model field").

Fig. 137: The "Add Parameter" window

  • Select the structure type as "Scalar/Array/Curve/Map" from the drop-down menu. Based on the data type selected, the fields below are displayed in the GUI.
Structure data type Fields in the UI
Scalar/Array Model field
Curve Model field, Model Breakpoint1
Map Model field, Model Breakpoint1, Model Breakpoint2
  • You can type and search the model field by clicking the drop-down button () from the "All model fields" in the "Model field".
  • The "Preview" displays the preview of the selected values.
  • Click Add to add a calibration variable.

Fig. 138: The "Add Parameter" window

To edit a variable

  • Click (Edit Variable) or right-click on a calibration variable and click .
  • The "Edit Parameter" window is opened.
  • The available parameters are automatically loaded.
  • Edit the values and the click Update.
  • Displays the existing values and the preview of the updated values.

Fig. 139: The "Edit Parameter" window

The maximize icon () in the "Preview/Existing" field is displayed for arrays, curves and maps which help to view the values clearly. When you maximize it, the data opens in a new window which also allows to scroll the values.

When you right-click on a calibration variable, the context menu is displayed which allows you to make the basic operation such as edit, cut, copy, paste, delete, and delete all.

Refer to Parameter Files Not Created for FMU After Migration if missing parameters in an FMU model after migration.

Refer to Calibration Variables in FMU for limitations of calibration variables in an FMU model.

Propose Variables for FMU Models

When you import a model, the parameters are automatically identified and generated in the "Calibration Variables" view based on the pre-defined patterns.

If those generated parameters are not fitting to your needs, you can configure it in the Settings menu (Auto Recognition Pattern). The "Propose Variables" suggests the list of possible maps and curves based on the patterns configured in the Settings menu. It also proposes, if any of the calibration variables are deleted manually. You can select the proposed variables from the list and create the variables.

This feature is applicable and visible only for the FMU models.

To propose calibration variable(s)

  • Click .
  • The "Propose Variables" dialog box is displayed.

Fig. 140: The "Propose Variables" dialog box

  • Check the checkbox(es) to select the variable(s). You can also select the checkbox on the top to select all variables at a time.
  • You can also search the variable(s) by clicking the down arrow () from the table. Type the variable name to filter it or check the provided checkbox to select the variable(s).
  • Click Create to create the variables.
