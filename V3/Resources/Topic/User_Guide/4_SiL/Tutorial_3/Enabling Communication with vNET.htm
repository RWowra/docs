Enabling Communication with VNET

  • Click on the CCodeModule model tab in the dashboard.

Fig. 31: CCodeModule model dashboard

  • Double-click on options.cmake file in the "Properties" view.
  • The options.cmake file is opened in the default text editor if the default text editor is mapped already. Else, a dialog box is displayed where you can choose a preferred text editor to open it.
  • Add the following library files and file references in the options.cmake as given below. These libraries are required to enable VNET communication.
Copy 
# additional libraries to be linked
set (EX_LIBS vNetInterface.lib)
 
# additional library paths to be linked
set (EX_LIBS_PATH "$ENV{COSYM_HOME}\\Simulator\\win64\\esse\\active\\link64\\")
# additional header file paths to be linked
set (EX_INC_PATH "$ENV{COSYM_HOME}\\Simulator\\win64\\esse\\active\\include\\")
  • External libraries and files are linked.
  • Double-click on CCodeModule.c file.
  • CCodeModule.c file is opened in the default text editor.
    The step-by-step instructions for VNET integration are detailed in the same file (<Demo data location>\Samples\SiL\Simple_CModule_vNet_CAN\Library\Control\CCodeModule\Sources\CCodeModule.c) itself. Refer the same to integrate your C-Code model with VNET.

In case, you need to create a project same as sample project, then copy the code from (<Demo data location>\Samples\SiL\Simple_CModule_vNet_CAN\Library\Control\CCodeModule\Sources\CCodeModule.c to CCodeModule.c file.

Note 

All signals in the C-Code model are physical signals and are of Double data type.
  • Create a network interface definition for your simulation unit

Create the modelControllers.config file as defined in the <COSYM Demo Data>\Samples\SiL\Simple_CModule_vNet_CAN\Library\Control\CCodeModule\Sources\modelControllers.config file and save the file in sources folder of the C-Code model. All logical names of vNet interfaces used by a model must be listed as attributes "logicalName" in the "controllers" section. These are the names used when calling VNET API mOpenPort().

Copy 
{
  "modelName": "CCodeModule",
  "controllersInfo": [
    {
      "controllers": [
        {
          "logicalName": "CCodeModule_Tx",
          "fullName": "CCodeModule_Tx",
          "busProperties": {
            "baudRate": "500000",
            "fastDataMultiplier": "0",
            "stepRate": "1000"
          },
          "controllerProperties": {
            "enableSelfReception": "false",
            "enableBusTermination": "false",
            "enableCANFDISOMode": "true"
          }
        },
        {
          "logicalName": "CCodeModule_Rx",
          "fullName": "CCodeModule_Rx",
          "busProperties": {
            "baudRate": "500000",
            "fastDataMultiplier": "0",
            "stepRate": "1000"
          },
          "controllerProperties": {
            "enableSelfReception": "false",
            "enableBusTermination": "false",
            "enableCANFDISOMode": "true"
          }
        }
      ],
      "busType": "CAN"
    }
  ]
}

 

The created file is shown below.

Fig. 32: modelControllers.config file in "Sources" folder

All the configuration (source code changes) mentioned above are to be updated in the options.cmake, CCodeModule.c, and modelControllers.config files, which can be seen in the <Demo data location>\Samples\SiL\Simple_CModule_vNet_CAN\Library\Control\CCodeModule\Sources\ folder.

Note 

All the C-Code models which use vNet API for virtual bus simulation should have a modelControllers.config file under \Sources folder of a model. vNet integration is possible only with this file.
