Interface

The Interface feature opens the Swagger editor to work with the COSYM REST APIs. You can use the interface to see all the available COSYM REST APIs and their details. It gives clear insight on how the API responds to parameters and options.

Follow the steps below to open the Swagger editor through COSYM GUI or headless mode where you can see the COSYM REST APIs and use it interactively.

To launch Swagger editor through COSYM GUI

  • Perform one of the following steps.
    • Go to Desktop and then double-click
      <Or>
    • Go to Windows icon → ETAS → ETAS COSYM and then click COSYM V3.4.1.
  • Read the "ETAS Safety Advice" document and click Acknowledged.
  • Splash screen is displayed and COSYM GUI is launched on your system.
  • Go to Help menu → API → Interface on the project screen or project dashboard.
  • Swagger editor is displayed.

To launch Swagger editor through headless mode

  • Go to Start menu on Windows and type "cmd".
  • Type cosym and press <Enter> on command prompt window.
  • Displays the commands which you can execute.
  • Type cosym --start --mode=headless.
  • Type cosym --acknowledgement=yes
  • COSYM is launched in the background.
  • Open any of the web browser and type http://localhost:8181/cosymserver/ and press <Enter> key.
  • Swagger editor is displayed.
Search

The search operation can be performed in the Swagger editor which helps to search the APIs.

To perform the search operation, follow the steps below

  • Click <Ctrl+F> key on your keyboard.
  • The "Find" window is displayed.
  • Type the text/API in the "Find" text box.
  • Check the relevant search options and click Next/Previous to perform the operation.

An Example on how to Use Swagger API

An example is given below on how to use the swagger API through COSYM.

This example demonstrates step by step process on how to connect the processes to tasks using "automap" API.

To open the Swagger editor through COSYM

  • Go to Help menu → API → Interface on the project screen or project dashboard.
  • The Swagger editor is displayed as shown below.

Fig. 199: COSYM REST APIs in Swagger editor

The REST APIs in the Swagger editor are available in the respective components. For eg., the "automap" API is available inside the OSTasks as shown below.

Fig. 200: APIs in the Swagger editor

To map the processes to the tasks, it involves to execute the respective APIs to perform the tasks below.

  • Create a project
  • Add the models
  • Retrieve the system id
  • Open a system
  • Add the models to a system
  • Execute an automap
  • Save a system
  • Retrieve all the mapped configuration

Each API contains request URL, description of an API and the parameters (payload) which needs to be passed to get the result.

To create a project

  • Go to Project Component and click on POST - /v1/projects
  • This API creates a COSYM project (*.csp) in the specified location.
  • Click Try it out.
  • Specify the project location, name, description, parameter file and a project state in the payload object.
  • Click Execute.
  • Returns a new project with the specified details and its id. Once a project is created, it also creates a default system.

After adding the model, you can make the OS configuration using automap API.

To execute the automap

  • Go to OS Tasks Component and click on POST - /v1/projects/{projectId}/systems/{systemId}/ostasks/automap.
  • Click Try it out.
  • Specify the project id, system id and update the incrementalAutomap to false.

  • The processes are mapped with the tasks.
