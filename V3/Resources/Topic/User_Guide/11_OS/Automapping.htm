Auto Mapping

The Auto mapping feature helps to configure the OS by mapping the processes into the relevant tasks.

The Auto mapping will be performed by COSYM with the following aspects. They are;

  • which process should go to which task, and
  • what is the order of the process within a task
Mapping of Processes to a Task
  • Process of the given type will be mapped to a task of the same type.
    For example, process of type "INIT" will be configured with "Init" task, processes of type "SOFTWARE" will be configured with "Software" task only.
  • If the process is of type "TIMER" with the given period value and delay, then it will be configured to a task with the same period and delay.
  • The mapping of processes depends on the Max. task(s) value specified in the Settings menu.
Order of the Processes in a Task

If a model 'A' depends on another model 'B', then processes of A are listed before the processes of B in a task.

The criteria which are mentioned below will also be taken care during Auto mapping.

  • Tasks in the hardware editor are coordinated with the tasks in the OS configuration after the Auto mapping.

If there is loop among two or more model instances. The loop will be resolved based on below rules:

  • Instance which is connected to most number of other instance will have highest weight.
  • If two or more instance have same highest number of connected instances, then among them instance which is connected by most instances will have highest weight.
  • If both above rules cannot resolve, then loop will be resolved based on the sorted order of path of the model instances.

To perform the mapping of processes to tasks using Auto mapping feature

  1. Click drop-down menu in the Deployment view.
  2. Click .
  3. A confirmation dialog box is displayed.

However, COSYM allows you to map the processes to the tasks as per your needs.
