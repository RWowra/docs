Configuring OS

The simulation is being executed on a target. To simulate a system, a target should be mapped with a system. You can use the default target or you can create a new target in the Deployment Management on the View menu. Now, the system has mapped with the SiL_target to run the simulation.

The order of how the simulation models will be configured and executed is given below in the next steps.

By default, the Deployment view is shown with Init and Exit tasks on the right side. The properties of the OS tasks can also be seen on the right side. On the left side, the simulation models are shown. When expanding the nodes of the models, it can be seen that each simulation model contains their processes, which can be mapped into the tasks on the right.

The "cosym_grt_Start_SMod_AllDataTypes_cosym" process contains the initialization code which should get executed before the actual simulation starts.

The "cosym_grt_MdlUpdates_SMod_AllDataTypes_cosym" and "cosym_grt_MdlOutputs_SMod_AllDataTypes_cosym" processes contain the code for execution. The task will run at 1 ms which is mentioned in the example below.

The "cosym_grt_Terminate_SMod_AllDataTypes_cosym" process contains the wrap-up code, which should get executed after simulation is completed.

Here, in this example, the processes are added to tasks manually by adding it.

To add a task

  • Click Deployment in the system tab.
  • The "Deployment" view is displayed.
  • Click Add task in the Task panel to create a new task.
  • The "Task details" dialog box is displayed.
  • Click Add to create a task.
  • A new task is created and displayed in the Task panel.
