Signal conversion

Signal conversion feature helps to modify the value of an Outport by applying certain rules to it. When conversion is applied, the new value is assigned to an Outport.

The values of the signal conversion are saved inside the <Project path>\Library\<Folder Name>\<Model Name>\Parameters\<Parameter name>.mpa file.

The Signal conversion can be applied only for the Active and Working Copy of a parameter file.

By default, COSYM creates a conversion model with the "OutControl" parameter value as 1, which means that the conversion model is enabled. Based on the type of conversion applied to signal/connection, the conversion formula applies.

Types of conversion

By default, Identity (no conversion) is applied. based on the requirement, the conversion values can be applied. The types of conversion are detailed below.

  • Identity
    No conversion (1:1) which means that no conversion rule gets applied. The value of the outport is transferred to the inport without any changes, i.e., 1:1 basis.
  • Linear
    Linear allows to specify factor + offset. This should have both the values to activate Apply button.
  • Curve
    It contains Curve(ltd) and Curve(ext).
  • Curve(ltd) allows to specify 2 parameters. They are:
    • List of inport values
    • List of outport values
  • The count of inport values and outport values should be equal. Outport values should be strictly monotone increasing or decreasing, i.e., n-1 < n or n-1 > n and the inport values can be arbitrary. Curve(ltd) should have 1 pair (input and output) values to activate the Apply button.
  • Curve(ext) allows to specify 8 parameters. They are:
    • List of input values
    • List of output values
    • xMax
    • xMin
    • yMax
    • yMin
    • Factor
    • Offset
  • The count of inport values and outport values should be equal. Outport values should be strictly monotone increasing or decreasing, i.e., n-1 < n or n-1 > n and the inport values can be arbitrary. Curve(ext) should have 2 pairs of values to activate Apply button.
  • You can add or delete the columns by clicking Add Column/Delete Column button. The add and delete operation occur at the end of a table.
  • Boolean
    If the specified threshold value gets exceeded, one of the specified highVal or lowVal value is given as output. This should have all the three values to activate the Apply button.
    • Low value (lowVal)
      Low value is assigned to the inport if the outport's value is less than the threshold.
    • High value (highVal)
      High value is assigned to the inport if the outport's value is equal to or greater than the threshold.
    • Threshold
      The threshold of the conversion rule, such that if (outport >= threshold) inport = highVal, else inport = lowVal.

Note 

Conversion cannot be applied on the port having String data type.

Apply signal conversion

Follow the steps below to apply the signal conversion rule:

  • Click on the respective tab in the Parameters view in a conversion model instance.

Fig. 149: Conversion tabs in a Parameter view

  1. Update the values.
  2. Click Apply.
    The values are applied.
  • Click Save to save the modified changes.
