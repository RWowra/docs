FMI 2.0 Support Details

Below is the list of FMI 2.0 attributes and the table also describes which attributes are supported or used in simulation.

Attribute

 

 Attribute Details

 

 Support in simulation
ModelIdentifier

 
  • For static library or source code, the modelidentifier is used as a prefix for fmi functions.
  • For DLL/SharedObject , modelidentifier is the name of DLL/Sharedobject.

 
  • FMU with static library or source code is not supported.
  • FMU with DLL/SharedObject is supported and modelidentifier is used as a name of DLL/Sharedobject.
needsExecutionTool

 

If it is true, a tool is needed to execute the model. FMU contains the communication to this tool.

 

 Not supported currently
CanHandleVariable CommunicatoinStepSize

 

The slave can handle variable step size. Communication step size (passed as a parameter to fmi2DoStep) is not constant for each call.

 

Not used. Communication step size is always constant.

canInterpolateInputs

 

The slave can interpolate continuous inputs.

 

Not supported currently
maxOutput DerivativeOrder

 

 The slave can provide derivatives of outputs with maximum order.

 

 Not used
canRunAsynchronously

 

 This flag describes the ability to carry out the fmi2DoStep call asynchronously.

 

 Not used
canBeInstantiated OnlyOncePerProcess

 

 This flag indicates cases where only one instance per FMU is possible.

 

 Not supported
canNotUseMemory ManagementFunctions

 

 If it is true, the slave uses its own functions for memory allocation and freeing only.

 

 Supported
canGetAndSetFMUstate

 

 If it is true, the environment can inquire the internal FMU state and restore it.

 

 Not supported

provides DirectionalDerivative

 

If it is true, the directional

derivative of the equations

at communication points

can be computed with

fmi2GetDirectionalDerivative

 

Not used

Co-simulation

 

FMU is based on "FMI for Co-simulation"

 

 Only Co-simulation.

UnitDefinitions

 

 Global list of unit and display unit definitions .

 

 Not used

TypeDefinitions

 

 Global list of type definitions that are used in "ModelVariables"

 

 Not used

DefaultExperiment

 

 Provides default settings for the integrator, such as stop time and relative tolerance.

 

 Not used
LogCategories

 

 Global list of log categories that can be set to define the log information which is supported from the FMU.

 

Infrastructure code is written and supported with the following limitations.

  • conversion of Value Reference to Variable name is not implemented
  • Log Category needs to be tested thoroughly

VendorAnnotations

 

 Additional data that a vendor might want to store and that other vendors might ignore.

 

 Not used

Model Variables

 

 The central FMU data structure defining all variables of the FMU that are visible/ accessible via the FMU functions.

 

  

ModelVariables -> ScalarVariable -> Casuality

 

Parameter, CalculatedParameter, input, output, local, independent Parameter, input, output, local are supported.

 

  

ModelVariables -> ScalarVariable -> Variability

 

Constant, fixed, tunable, discrete, continuous. Now all variables are treated as "tunable".

 

  

ModelVariables -> ScalarVariable -> initial

 

Exact, approx.., calculated

 

 Supports Only Exact

ModelStructure

 

 Defines structure of the model. Especially the ordered list of outputs, continuous time states and initial unknowns are defined here.

 

 Not used
