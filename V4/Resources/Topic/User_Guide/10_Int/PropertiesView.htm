Setting up the Properties of a System

The Properties view consists of 4 major sections. They are:

  • General Properties
  • Documentation
  • XCP Connection
  • Artifacts

General Properties

In the 'General Properties" section, displays the name of a system and allows to make the operations with respect to a label file.

Label File

The "Label File" helps to change the name of an entity such as inport/outport names, measurement variable names, and virtual connections based on the imported label file of *.yaml format. The applied port names will be reflected in the "Connections" view of a system.

A label file can be imported into a system. The applicable entities of the specified model instances in a label file are applied in that system only. It supports for a single model instance and also for batch processing.

You can specify the required details in the label file and you can import that label file in a system. The values are applied in the model instance(s) if that particular model instance(s) name is(are) mentioned in the label file. The values are displayed only when you enable the label file. The code snippet below shows an example for a label file.

Copy 
label_regex:
-
    model_instance_name: MiL_Three_Cyl_7_ECU,MiL_Three_Cyl_7_Plant # Model Instance Name
    inports: # Regex for Inports
     - '.*ICE_facLam.* -> ICE_fuel'
     - '.*ICE_facLoad.* -> ICE_Load'
     - '.*ICE_nEng.* -> ICE_Engine'
     - '.*ECU_IS_iaDes1.* -> ECU_Diesel1'
     - '.*ECU_IS_iaDes2.* -> ECU_Diesel2'
     - '.*ECU_IS_iaDes3.* -> ECU_Diesel3'
     - '(.*)ECU_FS_facFuelDes(.*) -> $1ECU_Fuel_Diesel$2'
     - '.*ECU_AS.* -> ECU_Throttle'
    outports: # Regex for Outports
     - '.*ICE_facLam.* -> ICE_fuel'
     - '.*ICE_facLoad.* -> ICE_Load'
     - '.*ICE_nEng.* -> ICE_Engine'
     - '.*ECU_IS_iaDes1.* -> ECU_Diesel1'
     - '.*ECU_IS_iaDes2.* -> ECU_Diesel2'
     - '.*ECU_IS_iaDes3.* -> ECU_Diesel3'
     - '(.*)ECU_FS_facFuelDes(.*) -> $1ECU_Fuel_Diesel$2'
     - '.*ECU_AS.* -> ECU_Throttle'
    parameters: # Regex for Parameters
     - '.*ICE_facLam.* -> ICE_$0_parameter'
     - '.*ICE_facLoad.* -> ICE_Load'
     - '.*ECU.* -> $0_ECU_parameter'
     - '.*Ecu.* -> $0_ECU_parameter'
     - '(.*)(Map)(.*) -> $2_MAP'
     - '(.*)(CUR)(.*) -> $1CURVE$2'
     - '.*table.* -> table'
     - '(.*)(bp)(.*) -> breakpoint'
     - '.*Gain.* -> Gain'
     - '.*HitCrossing(.*) -> $1_HitCrossing'
     - '(.*)(deg)(.*) -> $2degree'
    measures: # Regex for Measures
     - '.*ICE.* -> $0_measure'
     - '(.*)ECU(.*) -> $1_ECU_measure_$2'
     - '.*Ecu.* -> ECU'
     - '.*MAP.* -> MAP'
     - '.*table.* -> table'
     - '(.*)bp(.*) -> $1breakpoint$2'
     - '(.*)Gain(.*) -> Gain_$2'
     - '.*HitCrossing(.*) -> $1_HitCrossing'
     - '(.*)deg(.*) -> $1degree$2'
     - '.*KLAF.* -> KLAF_measure'
     - '(.*)ms_IC(.*) -> $1measure_IC$2'
     - '(.*)LoadPulsefortesting(.*) -> $1LoadPulse$2'

Note 

To import a label file

  • Click on the "General Properties" in the "Properties" view of a system.
  • A Windows Explorer is opened to select a label file.
  • Select the label file and click Open.
  • Click Import on the "Import Label File" dialog box.
  • Click Save to save the system.

To enable the imported label file

  • Check the "Enable" checkbox in the "General Properties" view. The checkbox is enabled only when the label file is imported.

Once the label file is imported and enabled, the respective parameters are reflected in the respective model instance(s).

To export a label file

  • Click on the "General Properties" view.
  • The "Export" dialog box is displayed.
  • Select the location to export a file and click Export.

Documentation

The "Documentation" section helps to write documentation for a system. It contains a text editor which helps to write documentation for a system. In the Documentation view, it allows to apply the content styling such as change the font family/type/size/color, apply the heading levels, superscript/subscript, text alignment, text indentation, bullet points, and many more. You can also read/edit the already available document in a system.

To add the contents

  • Go to text editor section in the Properties view.
  • Type the contents in the editor. It allows you to apply the documentation styles for the texts.
  • Click Save to save the changes.

XCP Connection

The XCP Connection section helps to enable the XCP Configuration during simulation, the enabled XCP connection connects INCA to a system.

In this section, it allows to perform the following operations:

  • Enable the XCP configuration
  • You should have a valid license. Check the checkbox to enable.
  • Change the IP address
  • Update the IP address of the XCP connection in the IP address field
  • Assign a task to an event
  • You can assign the "TIMER" task to an event when XCP configuration is enabled.

To assign a task to an event

  • Click Show advanced settings.
  • By default, the events are assigned with the DAQs (Data Acquisition) which cannot be changed and there are 5 events available.
  • Assign a task to an event by selecting it from the drop-down menu. The timer task which has the lesser period is considered as the fastest task which is assigned to Event 1.
  • Click Save on the tool bar to save the changes.

Artifacts

The arbitrary files can be added to a system which can be deployed to a target. If you want to include specific configuration files (e.g. *.xs) to a system or additional files/folders that are to be used can be added in the Artifacts section in the Properties view.

To add the artifacts

  • Click .
  • The "Import Artifacts" dialog box is displayed.
  • Specify the path and the file name to add to the list in the "Location" field or clock Browse to select it. You can select one or more files/folders.
  • The selected file(s) is(are) added to the list.
  • Click Import.
  • The files/folders in the list are imported.

Fig. 178: Import Artifacts dialog box

You can also filter the artifacts which are added. It can be done by typing the file name in the field.

To open an artifact

  • Double-click on a file.
  • <Or>
  • Right-click on an artifact, click Open File.

To open the location of a file/folder

  • Right-click on an artifact, click Open File Location.

While opening a file, if the file format is unsupported, then a dialog box is displayed to choose the corresponding application to open it.

To delete an artifact

  • Select an artifact and click .
  • <Or>
  • Right-click on an artifact, click Delete.

To delete multiple files

  • Select the files that you want to delete and right-click.
  • A context menu is displayed.
  • Click to delete all the files in the Artifacts view.
