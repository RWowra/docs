System Tab

A system tab is an environment where you can instantiate several models. Connections can be done manually by either dragging and dropping the ports or automatically where COSYM assists you in connecting the models. You can also import and export the connections.

The panels and table columns available in different views in a system tab are resizable. This will be indicated when you hover over lines with the mouse and cursor is changed to the resizable icon (). When you switch to another tab/view, the resized panel/table columns reset to defaults. It will not be persisted when you close and reopen a system.

The data in a table in the connections view and deployment view are alphabetically sorted by default.

When you create a new system or open a system by double-clicking on it, a system tab is opened in the project dashboard.

A target can be mapped and OS can be configured manually or automatically in order to prepare the system for code generation and run the simulation.

One or more systems can be opened in the project dashboard.

In a system tab, you can perform the following operations:

  • Connect the model instances
  • Apply signal conversion
  • Add model instances into a system
  • Map a target to a system
  • Configure OS to generate code and run simulation
  • Add the arbitrary files to a system which can be deployed to a target
  • Define parameter values for model instances
  • Enable/disable and export/import a label file
  • Add documentation
  • Enable and configure XCP connection, XCP session(s), and A2L file(s)

These operations can be done in six different views which are listed below:

  • Diagram View
  • Connections View
  • Deployment View
  • Parameter Assistant View
  • Properties View
  • XCP Config View
