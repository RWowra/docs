Working with INCA

Open the exported A2L file in COSYM into INCA to calibrate the values.

To open the exported A2L file in COSYM

  • Create a Root folder by right-clicking on the white background area of the Database Object.
  • A context menu is displayed.
  • Click Add → Add Top Folder.
  • A Root folder is created.
  • Right-Click on Root folder and click Add → ECU-Project (A2L).
  • An explorer is displayed to select the A2L file.
  • Select the exported A2L file in COSYM and click Open.
  • An explorer is displayed to select the *.hex file.
  • Click Cancel to ignore this step.

To create the Workspace

  • Right-click on the opened A2L file and click Add → Workspace.
  • A workspace is created. Each workspace can contain multiple devices.

Fig. 64: A workspace created in INCA

To add a device

  • Click .
  • A window is opened to add a device.
  • Select the A2L file.
  • When you select the A2L file, the INCA understands this selected A2L file which contains the XCP device.
  • Click OK.

Fig. 65: Adding devices

In the Ethernet system, it contains XCP and XCPSimTime. XCP contains the real time data and XCPSimTime contains the simulation time data.

  • Select XCPSimTime and click OK.
  • The hardware is added to the workspace.

To configure the hardware

  • Select the XCPSimTime hardware and click .

Fig. 66: Hardware configuration

To initialize the hardware

Initialization can be done when the models are downloaded in ETAS EE.

  • Click on the INCA toolbar.
  • INCA connects to the system simulation.
  • Select the "Action" Upload and click Do it.
  • Gives a warning message that it is not a real ECU. Click Yes to continue.
  • Select the location to upload. If the location is not specified, it will be saved in the Empty Dataitem_1.
  • Close the window.

Fig. 67: Connecting to system simulation

To select the variables

Initialization can be done when the models are downloaded in ETAS EE.

  • Click (Experiment Environment) on the INCA toolbar.
  • Click (Variable Selection).
  • "Variable Selection" window is displayed which displays all the variables in a model.
