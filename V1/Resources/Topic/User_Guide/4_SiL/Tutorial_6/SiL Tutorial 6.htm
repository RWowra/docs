Simple SiL Simulation Automated with REST APIs and ASAM XIL APIs

This demo enables a first hands-on experience of illustrating an idle speed controller of a 3-Cylinder gasoline engine using adaptive time simulation (ATS) which is conducted using COSYM REST APIs and ASAM XIL APIs. To demonstrate the simulation, it uses the client code which is generated in Python.

The two sample models are used which are listed below and these models are shipped with the COSYM installer.

  • Engine model
  • Engine controller model

Note 

It is recommended to execute the Simple SiL Simulation via COSYM GUI. The steps involved in this tutorial are same as Simple SiL Simulation where as COSYM REST APIs and ASAM XiL APIs are used here to execute it.

REST_ASAMXIL_APIs Demo

The demo is available at <COSYM demo data location>\REST_ASAMXIL_APIs.zip, if you have chosen the default location for COSYM demo data during COSYM installation. In case you have selected a different location for demo data while installing COSYM, then the REST_ASAMXIL_APIs.zip can be found in the location specified by you.

Demo models

The demo models are available in the COSYM demo data location (COSYM Demo Data).

Unzip the Samples.zip file which is available in the COSYM demo data. The simulation models are located at <Demo data location>\Samples\SiL\ATSModels\FMU sub-folder in the unzipped Samples folder.

Available Files in REST_ASAMXIL_APIs.zip File

When you unzip the REST_ASAMXIL_APIs.zip file, the following files are present and are described below:

  • DataProvider.json
    This file contains the data like project name, project destination folder, system, target name. And also the models folder that needs to be imported, and signals that needs to be logged, etc.
  • The models to be imported will consider the 'COSYM_HOME' from Environment Variable to get ETAS demo data location and the path given in *.json file
  • COSYM_REST_APIs.py ---> Class file containing multiple methods/functions of COSYM REST APIS that helps to run end to end workflow.
  • ASAM_XIL_APIs.py ---> Class file containing multiple methods/functions of ASAMXIL APIS that helps to run end to end workflow.
  • EndToEnd_WorkFlow.py ---> It covers the end to end workflow of COSYM REST APIs and ASAM XIL APIs as mentioned in below steps.
  • 3Cyl_ATS_Connections.xml ---> It is used to make the necessary connections.

Prerequisites

  • COSYM installed and it should be closed
  • Demo models which are mentioned above (Demo models).
  • Update the COSYM_REST_APIs.py and DataProvider.json files, if you had changed the COSYM Demo Data location during installation.
    • Go to the <Demo data location>\REST_ASAMXIL_APIs\ folder
    • Update the new demo data location by removing the assigned value for the variables mentioned below and update with the new path
      • ETAS_Data_Path in the COSYM_REST_APIs.py file (Line no. 42)
      • Connection_FilePath in the DataProvider.json file (Line no. 23)
  • Python 3.71 Verified with Python 3.7 or above is installed including pip
    • COSYM ships the Python 3.7 and it is available at <COSYM Installion Location>\Software\Python37
    • If Python was already installed by you, then it is available at <Selected location for Installation>\Python\PythonXX where XX is the Python version number
      The pip is available inside the \Scripts folder in any of the above locations
    • Make sure that you have added "PYTHONPATH" in the Environment Variable with the installed Python path
    • pip must be able to establish a connection to the package repositories. If any issues, refer to pip Related Issues
    • Go to the \Scripts folder and type 'cmd'
    • Execute the commands below in the opened command prompt.2 Already installed with the COSYM shipped Python package
      Execute the commands below to get the *.six, *.urllib3, pythonnet, and certifi packages which will be used while running the python EndToEnd_WorkFlow.py script
      • pip install six
      • pip install urllib3
      • pip install pythonnet
      • pip install certifi
  • Run the ASAM AE XIL API Standard Assemblies 2.1.0.msi file from <Installation DVD 2>\DATA\MISC\ASAM HiL and XiL APIs to install ASAM XIL
  • The path of the ASAM XIL.dlls i.e., ASAM.XIL.Implementation.Testbench.dll, ASAM.XIL.Implementation.TestbenchFactory.dll, ASAM.XIL.Interfaces.dll from the installed location have to be added into the Environment Variable under the name 'ASAM_XIL_DLLS'
  • The swagger_client folder which is created during the generation of Python Client Code should be placed in the folder where EndToEnd_WorkFlow.py file is placed
  • The COSYM_REST_APIs.py, ASAM_XIL_APIs.py, EndToEnd_WorkFlow.py, and DataProvider.json should all be placed in the same folder

Generating Client Code

  • Launch COSYM Product if it is not opened earlier. Refer to Launching COSYM.
  • Go to Help menu → API → Generate client... on the project screen.
  • "Client generation" dialog box is displayed.
  • Select the type, "Python" from drop-down menu as language of the client code which is to be generated.
  • Specify the path to export the generated client code.

    For example, specify "C:\Temp" in the Location field.
  • Click Export.
  • Client code is exported into "C:\Temp\" folder.
    This generated swagger_client folder needs to be placed in the same folder where EndToEnd_WorkFlow.py file is present.

Run Demo

To run the demo

  • Go to the unzipped REST_ASAMXIL_APIs folder.
  • Run the command prompt from the same location in the Windows Explorer.
  • Run the command, python EndToEnd_WorkFlow.py in the opened command prompt.

Tutorial Steps

Once you run the command above, it executes the end to end workflow of Simple SiL Simulation. It starts from the creation of a COSYM project until simulation process disconnection.

The same is described in the Simple SiL Simulation and please refer to Simulation Steps for ATS use case for the complete details.

The steps covered by COSYM REST APIs are listed below:

  • Create a project
  • Create a system
  • Create a SiL target
  • Map a target to a system
  • Import models
  • Import connections
  • Auto Mapping
  • Code generation, build and run ETAS EE

The steps covered by ASAM XIL APIs are listed below:

  • Download
  • Enable Data Logger and Start Data Logger
  • Start Simulation
  • Stop Simulation after a while
  • Disconnect

Refer to To view the simulation result for viewing the simulation result.
