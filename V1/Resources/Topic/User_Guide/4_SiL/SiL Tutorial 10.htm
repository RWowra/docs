Simple SiL Simulation Automated with REST APIs

This demo enables a first hands-on experience of illustrating the simulation of 3-cylinder gasoline engine model using COSYM REST APIs. To demonstrate the simulation, it uses the client code which is generated in C-sharp. The two sample models are used which are listed below and these models are shipped with the COSYM installer.

  • Engine model
  • Engine controller model

REST API Demo

The REST API demo is available at <COSYM demo data location>\COSYM_REST_Demo.zip, if you have chosen the default location for COSYM demo data during COSYM installation. In case you have selected a different location for demo data while installing COSYM, then the COSYM_REST_Demo.zip can be found in the location specified by you.

Prerequisites to Run REST API Demo

  • Launch COSYM
  • Visual Studio 2017 or Visual Studio 2019

Run REST API Demo

It is necessary to generate the client code to run this demo.

To generate the client code

  • Launch COSYM Product if it is not opened earlier. Refer to Launching COSYM.
  • Go to Help menu → API → Generate client... on the project screen.
  • The "Client generation" dialog box is displayed.
  • Select the type, "Csharp" from drop-down menu as language of the client code which is to be generated.
  • Specify the path to export the generated client code.
    For example, specify "C:\Temp" in the Location field.
  • Click Export.
  • Client code is exported into "C:\Temp\src\IO.Swagger\IO.Swagger.csproj" file.

To run the REST API demo

  • Double-click on COSYM_REST_Demo.sln file which can be found when you unzip the COSYM_REST_Demo.zip file.
  • It loads the demo in the Visual Studio application.
  • Double-click Program.cs from the "Solution Explorer".

The Program.cs file contains the code for cleanup activities, create a project, import the simulation models, open a system, add the hardware and the simulation models into a system, create the connections, configure the OS, generate the code, build a system, and open ETAS Experiment Environment (ETAS EE) to view the simulation results.

Fig. 23: Program.cs file

  • Right-click on the "COSYM_REST_Demo" solution and click Add → Existing project....
  • A windows explorer is displayed to select the generated code for a client.
  • Select the generated client code.
    In this demo example, it is exported into "C:\Temp\src\IO.Swagger\IO.Swagger.csproj" file as shown above.
  • Right-click on the "COSYM_REST_Demo" project and click Add → Reference....
  • The "Reference Manager" is displayed.

Fig. 24: Adding Reference

  • Select the IO.Swagger file from the Reference Manager and click OK.

Fig. 25: Reference Manager

  • Right-click on the "COSYM_REST_Demo" solution and click Clean Solution.
  • Right-click on the "COSYM_REST_Demo" solution and click Rebuild Solution.
  • Restores all the necessary packages and rebuilds the solution.
  • Click Start on the toolbar in the Visual Studio application.
  • Exposes all the COSYM APIs which are written in the "COSYM_REST_Demo" project to demonstrate the 3-Cylinder example.
  • The ETAS EE Product is launched to view the simulation result. Refer to Viewing the Simulation Result .

The default port used for the COSYM application is 8181. If the different port is used, then you need to change the value of DOMAIN in the Settings.settings file which is available in the Solution Explorer in the Visual Studio.
