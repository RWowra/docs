Enabling Communication with VNET

To enable the communication with VNET

  • Click on the C-Code model (Lin_Master_Slave) tab in the dashboard.
  • The "Properties" view is displayed.
  • Define a default text editor for the options.cmake file.
  • The options.cmake file is opened in the default text editor if the default text editor is mapped already. Else, a dialog box is displayed where you can choose a preferred text editor to open it.
  • Double-click on options.cmake file in the "Source file" section.
  • The options.cmake file is opened in the default text editor if the default text editor is mapped already. Else, a dialog box is displayed where you can choose a text editor to open it.
  • Add the following library files and modify the library reference as given below.
    Please note the VNET integration works with these reference files. These libraries are required to enable VNET communication.
Copy 
# additional libraries to be linked
set (EX_LIBS vNetInterface.lib)
  
# additional library paths to be linked
set (EX_LIBS_PATH "$ENV{COSYM_HOME}\\Simulator\\win64\\esse\\active\\link64\\")
  
# additional header file paths to be linked
set (EX_INC_PATH "$ENV{COSYM_HOME}\\Simulator\\win64\\esse\\active\\include\\")
  • External libraries and files are linked.
  • Double-click on Lin_Master_Slave.c file.
  • The Lin_Master_Slave.c file is opened in the default text editor.
    The step-by-step instructions for VNET integration are detailed in the same file (<Demo data location>\Samples\SiL\Simple_CModule_vNet_LIN\Library\Control\Lin_Master_Slave\Sources\Lin_Master_Slave.c) itself. Refer to the same to integrate your C-Code model with VNET.

In case, you need to create a project same as sample project, then copy the code from (<Demo data location>\Samples\SiL\Simple_CModule_vNet_LIN\Library\Control\CCodeModule\Sources\Lin_Master_Slave.c to your Lin_Master_Slave.c file.

Note 

All signals in the C-Code model are physical signals and are of Double data type.
  • Create a network interface definition for your simulation unit.

Create the modelControllers.config file as defined in the <COSYM Demo Data>\Samples\SiL\Simple_CModule_vNet_LIN\Library\Control\Lin_Master_Slave\Sources\modelControllers.config file and save the file in sources folder of the C-Code model with appropriate logical names in the attributes "logicalName" in the section "controllers". These are the names used when calling VNET API mOpenPort().

Specify the bus and the controller properties.

Copy 
{
  "modelName": "LinMasterSlave",
  "controllersInfo": [
    {
      "controllers": [
        {
          "logicalName": "LIN:0",
          "fullName": "LIN:0",
          "busProperties": {
            "baudRate": "19200",
            "stepRate": "1000"
          },
          "controllerProperties": {
            "role": "master"
          }
        },
        {
          "logicalName": "LIN:1",
          "fullName": "LIN:1",
          "busProperties": {
            "baudRate": "19200",
            "stepRate": "1000"
          },
          "controllerProperties": {
            "role": "slave"
          }
        }
      ],
      "busType": "LIN"
    }
  ]
}

 

All the configuration (source code changes) mentioned above are to be updated in the options.cmake, Lin_Master_Slave.c, and modelControllers.config files, which can be seen in the <Demo data location>\Samples\SiL\Simple_CModule_vNet_LIN\Library\Control\Lin_Master_Slave\Sources\ folder.

Note 

All the C-Code models which use VNet API for virtual bus simulation should have a modelControllers.config file under \Sources folder of a model. VNet integration is possible only with this file.
