Communicating with Bus in ETAS EE

When you click Build and Run in COSYM, a workspace is opened in ETAS EE.

If the ETAS EE does not show any ports created in COSYM, then follow the steps below.

  • Expand SIL under "All Asam" tab (Left side panel below the Workspace Elements).
  • Select the required ports which are to be viewed, then drag and drop.
    The ports can be grouped together by selecting the related ports and dragging them together or they can be dragged and dropped under already created groups.
  • Rename the groups according to your requirement.

Fig. 34: Workspace in EE

This configuration can be done in Bus Config feature in ETAS EE.

To make the bus config

  • Click Bus Config on the ETAS EE toolbar.
    <Or>
  • Go to Experiment menu → Bus Config on ETAS EE.
  • A dialog box is displayed to make the bus configuration.

Fig. 35: Bus config feature in ETAS EE

The logical controller names are displayed on the left panel and the bus and controller properties are displayed on the right panel.

  • Configure the bus properties and the controller properties on the right panel as given in the Tab. 6.
Logical controllers Buses
CCodeModule_TX Bus1
CCodeModule_RX Bus1

Tab. 6: Bus and controller mappings

  • Click OK to make the configuration.
  • The configurations made are saved.

Now, you can download the models and view the simulation result.

To download the models in ETAS EE

  • Go to Experiment menu in ETAS EE and click Download → SiL. Wait for the message, Model Download completed successfully logged in the "Application Log" window in ETAS EE.

To simulate a system

  • Set the simulation end time as 10 seconds on the "Adaptive time" field in the toolbar of ETAS Experiment Environment (ETAS EE).
  • Click Start Simulation → SiL in the Experiment menu.
  • Then a message Simulation started is displayed in the Application Log.
    Ends the simulation after 10 seconds.
  • Click Disconnect to disconnect the simulation process.

Fig. 36: Data send and receive during simulation

When this simulation is run, the data sent by C-Code will appear under CCodeModule_Tx and received data will appear under CCodeModule_Rx.
