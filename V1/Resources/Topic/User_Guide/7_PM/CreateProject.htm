Creating a Project

The Create new project feature is used to create a new COSYM project in the specified location. The COSYM project is stored as *.csp extension.

You can create a system, folders and sub-folders within a project. COSYM project allows to integrate the models, configure the OS, and simulate the systems.

To create a new project

  1. Perform one of the following steps:
    1. Click in the project screen.
      <Or>
    2. Go to File menu on the Project → New → Project.
  2. The "Create new project" dialog box is displayed.

Fig. 85: Create new project dialog box

  1. Type "Name" of a project.
  2. If the given name already exists in the specified location, an error message appears.

Note 
  • While creating or renaming a project, only the windows valid special characters are allowed:
    For example, characters like \, /, :, *, ?, ”, <, >, and | are not allowed.
  • Space in a project name or project path is not allowed.
  • Do not use the reserved names:
    For example, CON, PRN, AUX, NUL, COM1, LPT2, NUL.txt, etc.
  • Refer to the Microsoft Windows documentation for more details on naming.
  1. Specify "Location" to store a project or click Browse to select the location.
  2. Type "Description" of a project.
  3. You can describe about a project.
  4. Select the format of a parameter file format from the "Parameter File Format" drop-down menu.
  5. The supported file formats are MPA and CDFX files. By default, MPA is selected.
  6. Click Create.
  7. A new project is displayed on the project list and it is created with <Project name>.csp format in the file system.

Note 

If you want to copy a project from one location to another from the file system, it might not work properly due to some technical issues. In such a case, use the Save as functionality (Saving As a Project) to save a project to another location.

Project List

When a new project is created, it is listed in the project list in the project screen. project list shows all the created COSYM projects and the related information of these projects.

The project list contains 3 columns, namely:

  • Project Name
    Name of a project
  • Path
    Path where the project is stored
  • Created On
    Project created date

Fig. 86: Project list

Project description

When you hover the mouse over a project, it will display the additional information about that project, if available.

Project sorting

By default, the projects are ordered alphabetically in the project list. You can also arrange the order of the projects by clicking on project sorting button.

COSYM Project in a File System

When a new project is created, the corresponding files and folders are also created in the project path (the location specified while creating a project) of the working directory which contains the following:

  • Hardware folder
  • Library folder
  • Systems folder and
  • <Project name>.csp file

.

Fig. 87: Folder structure of the project in the file system

Hardware folder contains the default target, SiL_target which is mapped with the default system "System". The created targets will be saved in the Hardware folder. The created/imported hardware models will be saved in the respective targets.

Library folder contains three default folders such as Control, Environment, VECU. These folders allows to create the C-Code model, folders/sub-folders. It allows to import all the models except the hardware models.

Fig. 88: Library in the file system

Systems folder contains all the created systems and its relevant files and folders. When a project is created, the default system "System" is also created which is mapped with default target "SiL_target". You can also create many more systems in a project. It provides facility to select and connect the models. It allows you to configure the OS.

Fig. 89: Systems in the file system
