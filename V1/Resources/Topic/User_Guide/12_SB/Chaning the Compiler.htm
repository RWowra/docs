Compilers

To build a SiL system, you need to have an appropriate compiler. The build process takes care the compilation internally.

COSYM supports the compilers mentioned below:

  • GCC 9.1.0 compiler of 32-bit/64-bit
    COSYM ships GCC 9.1.0 compiler along with the installer and this is used as a default compiler during build.
  • Visual Studio 2019
    The community edition can be used. Ensure that the licensing conditions are fulfilled.

Note 

If you are facing any issues related to the above packages, please contact ETAS Sales Team.

Changing the Compilers

It is possible to change the compiler to Visual Studio 2019 from the default GCC compiler as given below. Make sure that the model(s) in a system should has(have) at least one measurement variable and one calibration variable to change the compiler to Visual Studio 2019.

To change the compiler

  • Go to the <COSYMInstallation Path>\COSYM_KARAF\cosym\appdata\ATS_CodeGen_Resources\build\ folder.
  • Edit the BuildWinx64.bat file in Windows 64-bit systems BuildWinx86.bat file in Windows 32-bit systems in any of the text editor.
  • In the Fig. 193, the GCC compiler is set by default and the default settings are highlighted.

Fig. 193: Default GCC compiler settings

  • To set the Visual Studio as a compiler, follow the steps below.
    • Comment the first three lines
    • Add a new line after these three lines
    • set path=%COSYM_CMAKE_HOME%;%path%
    • Change the value of the variable COMPILER (Shown as line number 8 in the Fig. 194) to "Visual Studio 16 2019" -A <XX> where <XX> is either x64 or Win32.

Fig. 194: Visual Studio compiler settings
