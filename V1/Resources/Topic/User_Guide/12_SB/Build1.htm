Build

This feature is used to generate and build the code for a system.

This feature forces the build process even if there is no change in a system.

To generate and build the code

To generate and build the code, perform one of the following methods below.

  • Go to Project menu and click Build.
    A window is displayed to select a system to build it.

Fig. 188: A system selection

  • Select a system and click Build.
    <Or>
    Click Build → Build.
  • The "Code generation and build process" dialog box appears.

Fig. 189: The "Code generation and build process" dialog box

  • Displays the status of the build.
  • Displays the path of the folder where the cloud deployables are generated. This is not displayed if you disable the "Create cloud deployable" option in the Settings menu.

During build process, COSYM checks if a system contains network restbus model(s) in it. If necessary, COSYM updates those model(s) to reflect the modified changes.

Go to View menu → Messages to view the status of the build process and the total time which is taken for the build process.

If the selected system for the build process is not mapped to any target, then COSYM shows an error as shown below.

Fig. 190: An error message

In such a case, click Cancel to cancel the process. Then, map a target to a system to continue further. Refer to Deployment Management for more details.
