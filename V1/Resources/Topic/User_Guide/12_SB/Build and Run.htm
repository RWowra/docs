Build and Run

Build and Run feature generates and builds the code. Then it starts ETAS Experiment Environment (ETAS EE) to run simulation experiment and view the results.

If the system was built earlier, it directly opens ETAS EE.

About ETAS EE

The ETAS Experiment Environment (ETAS EE) is an ETAS product. ETAS EE is also installed during COSYM installation. Refer to "Online help" document which is embedded in the ETAS EE for more details on Experiment Environment. Also refer to Running the Simulation Experiment for more details.

Build the Code and Run ETAS EE

To build the Code and start the ETAS EE to run simulation experiment, perform one of the following methods below:

  • Click on the toolbar.
  •  <Or>
  • Go to Project menu and click Build and Run.
  • A window is displayed to select a system.

Fig. 191: A system selection

  • Select a system and click Build and Run.
    <Or>
    Right-click on a system.
  • A context menu is displayed.
  • Click Build → Build & run.
  • The Build process starts. The progress bar indicates the current action of the build process and it disappears at the end of the build process.

During build process, COSYM checks if a system contains network restbus model(s) in it. If necessary, COSYM updates those model(s) to reflect the modified changes.

After the successful of the build process, the code will be built and executable will be generated. It starts the ETAS EE to run simulation experiment and to view the results.

Go to View menu → Messages to view the status of the build process and the total time which is taken for the build process.

System Build for the Updated Source Code of the Models

The changes done to the source code of the models used in the system, will reflect in the build of the systems on the below scenarios:

  • If there are other changes also done to that system in COSYM and the system requires a save in COSYM. In this case, when you select Build & Run option, then build will be triggered and the changes done to the C-Code model will reflect.
  • If there are no other changes done to the system where the update of source code of the model update is part of it. In that case, you need to trigger a force build of the system using the Build button for the changes to reflect in the system.
