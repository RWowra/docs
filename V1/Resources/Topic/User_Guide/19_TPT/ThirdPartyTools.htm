Connecting 3rd Party Tools to COSYM Simulation

This feature allows 3rd party tools to connect to COSYM simulation using APIs which are exposed to clients through the TargetClient library.

With this feature clients can,

  • Control the simulation
  • Runtime modification/monitoring of signals
  • Sync read/write to other models based on simulation time
  • Have access to simulation states

Connecting External Model

External models can be a part of the COSYM simulation from 3rd party tools.

When you enable the provided checkbox in COSYM, then it connects a model to 3rd party tool for simulation.

To connect to the 3rd party tooling for simulation

  • Check the checkbox in the "Properties" tab of a model instance.
  • Click to view more details on how to connect a model to 3rd party tool.

Fig. 197: The "Execute externally" checkbox

The 3rd Party tool needs to implement the model using the APIs from TargetClient library for the simulation of the marked external model in COSYM.

During the download, you need to connect the external model. Download process waits for the external model to be connected to mark the successful completion. Simulation cannot be started without external model connection if a project contains an external model.

Refer to the snippet below for connecting the external model.

Copy 
static double xinput = 0.0;
static double xoutput = 0.0;

static double xcalibration = 10.0;
static double xmeasurement = 0.0;

void exportInitFunction() {
}
void exportExitFunction() {
}
void exportStepFunction(const char* functionName, uint64_t sim_time, int64_t raster_us) {
    if (strcmp(functionName, "Process1") == 0)
    {
        if (xinput <= 5000)
            xoutput = xinput + xcalibration;
    }
}

SetImportTimeout(-1); // wait till simulation is up
void* pmod = nullptr;
GetSilLoader("ExportModel", pmod);

//get no of models
int models_no = -1;
GetNumberOfModels(pmod, &models_no);

// get model handle by name
void* xtool = nullptr;
GetModelHandleByName(pmod, "ExportModel", xtool);

// define inport and outport locations
void* outports = nullptr;
GetSignalGroup(xtool, SGOutports, outports);
void* inports = nullptr;
GetSignalGroup(xtool, SGInports, inports);

SetSignalBufferWithName(outports, "Out1", &xoutput);
SetSignalBufferWithName(inports, "In1", &xinput);

SetVariableSetCallback(xtool, writeVariableCallback);
SetVariableGetCallback(xtool, readVariableCallback);

//register the step function
SetModelInitCallback(xtool, exportInitFunction);
SetModelStepCallback(xtool, exportStepFunction);
SetModelExitCallback(xtool, exportExitFunction);

RunLoaderAsync(pmod);

while (1)
{
    if (xoutput >= 5000) {
        break;
    }
}

CloseSilLoader(pmod);

 

Refer to the SiL_Python_API_Guide.pdf for more details and specification.

Use Case

The 3rd party tools such as test automation tools, debugging tools and simulation tools can use this feature to achieve co-simulation. An external model can be created and used for data exchange during the co-simulation.

Controlling Simulation

A 3rd party tool can use TargetClient APIs to control the simulation. By using the APIs, you can deploy, start, pause, and stop the simulation. Also, other features such as signal generator, data logging and parameterization etc. can be used.

Refer to the SiL_Python_API_Guide.pdf for more details and specification.

Along with the control simulation, you can connect the external model from the same tool. As a constraint, there can be only one tool for controlling the simulation.

Use Case

The 3rd party tools can use this feature for test automation and validation. A simulation can be deployed and controlled through APIs from 3rd party tools.

Connect to Running Simulation

The 3rd party tools can be connected to a running SiL simulation by using the APIs from TargetClient library. By providing the session ID of a running simulation, 3rd party tools can get the details of the simulation info, state, and read/write parameters. Also, other features such as Signal Generator, Data Logging and Parameterization etc. can be used.

Following is the snippet for connecting to running simulation,

Copy 
ConnectToRunningSiL(-1);
networkState state;
GetCurrentSimulationState(&state);
double time = -1;
GetSimulationTime(&time);

Sim_Info_Handle handle = 0;
LoadCurrentSimulationInfo(&handle);
int count = 0;
GetNumberOfVariables(handle, VarParam, &count);
GetVariableInfo(handle, VarParam, 0, &vinfo);

 

Refer to the SiL_Python_API_Guide.pdf for more details and specification.

Use Case

The 3rd party tools can use this feature to connect to a running SiL for test automation and validation with various scenarios. Simulation meta data can be requested using API and other features such as parameterization, signal generator and data logging can be performed. Multiple 3rd party tools can be connected to a running SiL at once.
