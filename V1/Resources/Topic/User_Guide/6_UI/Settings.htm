Settings

The Settings allows to make the changes in different levels in COSYM.

  • Product level
  • Project level
  • System level
  • Target level

Making Settings from the Project Screen

Settings from the project screen allows to make the product level settings. Here, you can change the configured port of the network restbus model and you can activate/deactivate the options Create cloud deployable and Build cleanup.

To update the product level settings

  • Click on the project screen.
  • The "Settings" dialog box is displayed.
  • Update the network restbus model if required. The default port used for the network restbus model is 9000.
  • Enable the setting Create cloud deployable. By default, it is disabled.
    Refer to Generating Cloud Deployables for COSYM for more details.
  • Disable the setting Build cleanup. By default, it is enabled.
    Refer to Build Cleanup for more details.
    • Click to view more details on build cleanup.
  • Click Update.

Making Settings from the View Menu

The "Settings" from the View menu in the project dashboard allows to make the product level, project level, system level, and target level settings.

To make the different levels of settings

  • Go to View menu and click .
  • The "Settings" dialog box is displayed.

Fig. 82: Settings from the View menu

It contains 4 different tabs to make the settings.

  • Product
  • Project
  • System
  • Target
Product

Allows to make the product level settings as described in the Making Settings from the Project Screen.

Project
  • Parameter File Format
    Select the format of the parameter file which can be selected during creation of a project.
  • Auto Recognition Pattern
    Configures auto recognition patterns for the FMU models and to create maps and curves for the FMU models. The configured patterns are applicable for all the FMU models in a project.
    Refer to Auto Recognition Pattern for more details.
System
  • System
    Choose the system which you want to change the settings by selecting the system name from the drop-down menu.
  • Max Priority
    Specify the maximum priority for the task. It takes the highest Priority value as 0 and the value ranges from 0 to 99. The smaller the value, the higher the priority.
  • Export user defined task with no process
    By default, COSYM exports the default tasks (Init, Exit) and the user defined task(s) having the process(s) in it. If any user defined task(s) is not having process in it, it will not export it. If you want to export those tasks having no processes in it as well, then enable the settings, "Export user defined task with no process" option.
  • Handling of unmapped processes when triggering build
    This setting is for choosing how to handle unmapped processes when triggering a build. The possible selections and their behavior are mentioned below: 
    • [0]: Automap
      If this option is selected and any unmapped processes are available in a system, then COSYM performs the incremental automapping for the unmapped processes and continues the build process automatically.
    • [1]: ignore
      If this option is selected, all unmapped processes are ignored and the build process starts without displaying the dialog box. But, a warning message is displayed to inform you that there are unmapped processes which are still available in a system and those processes will not be executed during the build and this could lead to a different simulation result.
    • [2]: Ask everytime
      By default this option is selected.
      When you start the build process for a system, a dialog box is displayed if any unmapped processes are available in that particular system. The dialog box prompts you either to map and build (same action as [0]:Automap) or to build without mapping (same action as [1]:ignore) the unmapped processes.
Target
  • Target
    Choose the target name from the drop-down menu.
  • Max.Task(s)
    Max. Task(s) takes an integer value starting from 1. The default value assigned by COSYM is 31.
    If the specified value is less than 1, then the error message is displayed during automapping.

Auto Recognition Pattern

As described in the "Auto Recognition Pattern" in Project, this feature helps to configure the auto recognition patterns (field and axis patterns) for the FMU models and helps to create the variables for the FMU models.

To configure auto recognition pattern

  • Click to make the configuration.
  • The "Pattern for Curve/Map mapping" dialog box is displayed.

Fig. 83: The "Pattern for Curve/Map mapping" dialog box

  • Click Update to save the configurations made.

To add field/axis patterns

  • Enter the new field/axis pattern in the provided text box and click .

To edit field/axis patterns

  • Click on the respective pattern which you want to edit.
  • The pattern is in editable mode.
  • Edit the pattern and click to update.

To delete field/axis patterns

  • Click on the respective pattern which you want to delete.

Note 

It is not possible to delete/edit the pre-defined patterns in a model.

To know more about patterns and view some examples

  • Click .

To change the priority of the patterns

The priority of the patterns are considered from top to bottom as they are visible in the UI. You can change the priority as below.

  • Select the pattern and hold the mouse button. Drag up or down to change the priority and release the mouse key.
