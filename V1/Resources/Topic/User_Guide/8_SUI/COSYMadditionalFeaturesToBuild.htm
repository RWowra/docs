COSYM_GRT Additional Options for Code Generation

You can also configure the few additional settings by checking the checkbox in "COSYM_GRT Additional Options" of the "Code Generation" tab.

Fig. 97: Additional options in the Gode Generation tab

Generating MdlOutputs and MdlUpdates in Place of rt_OneStep

Once you generate *.slxip for a model, its simulation involves execution of functions such as "initialize", "computation" (called recursively) and "terminate" available in the C files.

If this checkbox is checked, the "computation" step divides into 2 separate C functions called "MdlOutputs" and "MdlUpdates". Else, the computation step happens within a single function/process called "rt_Onestep".

Generating Substrate as Independent Processes

If this checkbox is checked for *.slxip generation in a model containing sub-parts that run at different sampling rates. The C-Code generated in the *.slxip contains different C function/process for each sample rate.

If it is not checked, the single computation step function is generated for all the different sampling rates.

Pack model artifacts

This feature helps to pack the model files (*.slxp, *.slx, *.mdl) along with its dependencies into a *.slxip file.

Use build only by default

When you check the checkbox "Use build only by default", builds only the source code, by default. During source code generation, "COSYM GRT Build Preferences" will be suppressed.

Fig. 98: COSYM GRT Build Preferences dialog box

Use build and import by Default

When you check the checkbox "Use build and import by default", it builds the source code and imports it into COSYM projects, by default. During source code generation, "COSYM GRT Build Preferences" will also be suppressed.

Check for LABCAR Ports

Check the checkbox "Check for LABCAR Ports" for LABCAR ports validation. By default, the checkbox is selected. You can deselect the checkbox when a model doesn't have LABCAR ports in it.
