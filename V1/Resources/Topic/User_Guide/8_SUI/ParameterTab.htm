Parameters View

The "Parameters" view which is available in a model dashboard shows all the parameter sets which are present in the selected model or model instance.

The properties of Parameters view are listed below:

  • The supported parameter file formats are *.mpa and *.cdfx. When *.dcm file is imported, it will be converted to *.mpa format internally and there is no loss of data during conversion.
  • You can import or export the parameter file to/from a model.
  • The properties of a parameter file is non-modifiable in the Parameters view of a model, but it can be modified in a model instance.
  • The default parameter set created by COSYM during creation is Inactive, by default. If you have specified a parameter file during creation, then the status is set to Active by default.
  • In a model, only 'Import' and 'Export' fields are enabled. The deleting and cloning of a parameter file can be performed only in a model instance. If a parameter file is imported, then deleting a parameter file is possible in a model.
  • Deleting of the active and original parameter file(s) is(are) not possible.
  • If a parameter value in a model is unknown and cannot be read out, it gets no value and is displayed as "" (empty string) when the default parameter file is created.
  • Page navigation feature is enabled when more parameters are available in a Parameters view. By default, 25 rows can be viewed in a page. It allows you to scroll-up or scroll-down in a page. The number of rows can be changed to 50/100/500 from the drop-down menu in the Page navigation. This drop-down menu appears when rows exceeds 25 onwards.

To view the Parameters view:

  • Open a model dashboard (To open a model dashboard for a model/model instance).
  • Click Parameters to navigate the Parameters view.
  • The fields in the Parameters view are displayed.

List of Parameter files

It shows the list of Parameter files available in the selected model or model instance. In a model, only the name of the parameter files are displayed.

Model instance contains the following details:

  • Name
    Displays the name of the parameter files.
  • Type
    Displays the types of a parameter files. The types are either Original or Working Copy.
  • State
    Displays the state of the parameter files. The state of the files are either Active or Inactive. By default, the state is Inactive.
  • Add to experiment
    The checkbox helps to select the parameter file to add it in the ETAS Experiment Environment (ETAS EE) for simulation. The added files may be selected in the future for simulation in ETAS EE.
    When the corresponding checkboxes of the parameter files are selected, those files are added to the experiment.
    During the simulation in ETAS EE, the Active parameter file is considered for the simulation.
    By default, the active parameter file is selected and added to the experiment. If you deselect the Active parameter file, then COSYM makes that parameter file Inactive.
Properties of a Parameter file

The properties listed below are displayed when you select a parameter set.

  • Name
    Name of a parameter file
  • Model path
    The path of a parameter
  • Value
    Value specified for the parameter
  • Max Value
    Maximum range of a value
  • Min Value
    Minimum range of a value
  • Type
    Data type of a parameter
  • Comment
    Comments if any

All the properties of a parameter file are non-modifiable in a model. But in a C-Code model, it can be modified in the Calibration Variables view.

Refer to Parameter Files in ETAS EE on how to get the parameter files in ETAS EE.

Importing a Parameter File

Import feature helps to import parameter file(s) into a model/model instance.

To import a parameter file

  • Click .
  • "Import parameter files" dialog box is displayed.
  • Specify the location of parameter file(s) to import or click Browse to select. The supported parameter files are .mpa, .cdfx, or .dcm files only.
  • Click Import.
Exporting a Parameter File

Export feature helps to export all the available parameter file(s) from a model/model instance.

Note 

Avoid spaces in the variable paths for any model type. The export of parameter files in DCM format will fail if there are spaces in the path.

To export a parameter file

  • Click .
  • The "Export parameters" dialog box is displayed.
  • Select the format of the parameter file to export it. By default, DCM file format is selected.
  • Specify the location of parameter file(s) to export or click Browse to select the location.
  • Click Export.
Cloning a Parameter File

Clone feature helps to create a replica of the selected parameter file(s). Clone will be enabled if one or more parameter files are selected.

To clone a parameter file

  • Select parameter file(s) and click .
  • Copy of "Inactive" parameter file(s) is(are) created.
Deleting a Parameter File

Delete feature deletes the selected parameter file(s).

To delete a parameter file

  • Select parameter file(s) and click .
