OS Configuration

Simulation works on the mechanism of OS scheduling. When a model is imported into a project, the relevant processes are automatically accepted in the OS configuration and assigned to suitable tasks. Once all the models are integrated, it is necessary to configure the simulation. It includes adding or removing of tasks, updating task properties, and assigning or removing of processes to or from Tasks panel. Hence, each OS task must be mapped with the processes of models.

While configuring the OS, "init" process should be mapped to "Init" task, "exit" process to "Exit" task only.

The order of processes in the task panel will be considered as the same that are added by you. You can also change the order by moving the processes up or down in the Deployment view. Data exchange between models happens in the same manner as in the order.

Auto mapping or incremental auto mapping features offered by COSYM can also perform the mapping of processes to relevant task.

Deployment view in a system tab helps to view the configurations and also allows you to configure the settings for simulation.

To view the Deployment view, click Deployment in a system tab.

The operations below can be done in Deployment view:

  • Target selection
  • Adding tasks
  • Adding hooks to a task
  • Mapping processes to tasks
  • Configuring OS
  • Import configuration
  • Export configuration
  • Auto mapping
  • Incremental Auto mapping
