Importing Connections

Import connections feature helps to import the connections. Connections from the *.xml file will be imported only for the model instances which are available in the composition. By default, it imports the real and the virtual connections. While importing, you can choose the checkbox to ignore the internal virtual connection.

When the virtual connections are imported, the external virtual connections are displayed in the Connections view of a system tab and the internal virtual connections of a model are displayed in the Virtual Connections view of a model dashboard.

To import a connection

  • Click in the Connections view.
    <Or>
  • Select a connection(s) in the Connection viewer and right-click.
  • A context menu is displayed. Click Import.
    <Or>
  • Right-click on the Connection viewer.
  • A context menu is displayed.
  • Click .
  • The "Import" dialog box is displayed as shown below.

Import dialog box

  • Specify the location or click Browse to select the location to import the connection(s).
  • Select the checkbox "Ignore internal virtual connections" to discard the internal virtual connections.
  • Click Import.
  • The "Replace or skip connection(s)" dialog box is displayed if the connection(s) in the selected file is already available.
  • Click either "Replace the connection(s) in the destination" to replace the existing connection(s) or "Skip these connection(s)" to ignore the connection(s).
  • The connection(s) from the selected file will be imported into a composition.

Note 

The connections will be created if its pins/ports are exactly same when you import a connection file of *.xml format.
