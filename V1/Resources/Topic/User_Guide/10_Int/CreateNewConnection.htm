Auto Connection

Auto connection can be done in the Connections view.

Sometimes, the connections are not clear which outport should be connected to which inport when a set of outports are selected. To overcome this issue, COSYM suggests connections based on the port names of the models/FMUs and their data types. All possible connections between the inport and outport of models are rated. The connection with the highest rating between an inport and an outport will be suggested to you in a tabular view. Based on the suggestions given by this method, you can decide whether the suggested connections have to be approved or rejected.

The page navigation feature is enabled when more connections are available in a table. By default, 25 rows can be viewed in a page. It allows you to scroll-up or scroll-down in a page. The number of rows can be changed to 50/100/500 from the drop-down menu in the page navigation. This drop-down menu appears when rows exceeds 25 onwards.

This feature is able to suggest up to a limit of 200,000 connections. Out of those suggestions, 700 connections can be approved and created at a time. Higher numbers of suggested connections may cause performance issues.

The Auto connection can be done in 2 ways as follows:

  • Connect identical ports
  • Propose connections

Connect identical ports

The algorithm used in the "Connect identical ports" are based on the port names. This happens only when the port names are identical. It considers the alphabetical order of the model names.

To add the models to a system

  • Switch to the Connections view.
  • Add the models into a system. See Adding to a System and Dragging and Dropping the Models for adding the models into a system.

  • Click in the Connections view.
  • Click the drop-down menu in the Create icon.

  • The context menu is displayed.

  • Based on the port names, the connection are made and displayed in the Connection viewer.

Else, there is no identical ports, then an error message is displayed.

Note 

If multiple inports and outports have same label names in the same location, then all the inports get connected to the last outport with the same label.

Propose connections

The algorithm used in the "Propose connections" are the port names. Similarity between these two port names (inport name and outport name) are considered as the rating for a particular connection. The rating from the "Jaro Winkler" and "Dice Coefficient" algorithms are compared and the highest value is set as the rating of the connections. Based on the port names, the algorithms display the match result. The algorithms propose the rate of accuracy as 100% if the number of characters are same. It still considers 100% accuracy even if the order (e.g., ICE_facLamExh_BdcX_EMS and ICE_EMS_BdcX_facLamExh) is interchanged.

The propose connection takes approximately 25-27 seconds for connecting 1000 ports (1000000+ connections).

To propose the connections

  • Add the models to a system (To add the models to a system).
  • Click .
  • Click the drop-down menu in the Create icon.
  • A context menu is displayed.
  • Click .
  • A new window is displayed with the suggested connections in a table format.
  • By default, the best rated connections are considered as connections.

Fig. 166: COSYM suggested connections/Preview table

  • Click Reject to reject the suggested connections.
  • Click Apply to make the connections or Cancel to cancel the suggested connections.

Note 

COSYM will not allow you to work until you close the Auto connection window by applying or canceling the suggested connections. You can apply the suggested connections by clicking Apply and cancel the suggested connections by clicking Cancel or click on the top right corner of the window or press <Esc> button on your keyboard.

Refer to Propose Connections for limitations.
