Configuring XCP

The "XCP Config" view in a system helps to generate the A2L file. The configuration of the A2L file is done by assigning the model instances and sessions with required IP addresses and internet protocols. The generated A2L file will be used for the COSYM-INCA workflow.

This is supported only for COSYM systems with SiL targets (SiL use-case).

The configuration can only be done when the XCP connection is enabled in the "Properties" view of a system and you have the valid license to use this feature. When this is enabled, Session 1 and A2L_File_1 are created by default. In addition, the A2L_File_1 is mapped to Session 1.

Fig. 179: XCP Config view

To enable the "XCP Config" view in the "Properties" view

  • Go to "Properties" view in a System tab.
  • Check the checkbox, "Enable" in the "XCP Connection" section.
  • Enables the "XCP Config" view in a System tab when you have a valid license. Refer to XCP Connection for more details.

To add a session in the "XCP Config" view

  • Click Add Session.
  • A new session is created.

The newly created sessions are appended with 1, 2, 3 and so on automatically. When the sessions are created, the ports are assigned with the IP addresses started from 5555 to the newly created sessions and it is increased by 1 for each newly created sessions. The TCP protocol is selected by default.

You can rename the session names by clicking on it and editing it. The port and protocol names can also be changed by clicking on the item directly. The port numbers range is allowed from 5555 to 65535 only. The UDP protocol can be assigned by selecting it from the drop-down menu.

To add an A2L file

  • Click Add A2L file.
  • The "A2L File" dialog box is displayed.
  • Type the name of an A2L file in the "Name" field.
  • Click any one session from the Session list which is mandatory.
  • Click Add A2L File.
  • An A2L file is created.

As soon as an A2L file is added, the status of the A2L file displays as "New".

To add a model instance(s) to an A2L file

The added model instance(s) to a system is displayed in the "Model Instances" panel of the A2L section.

  • Select one or more model instances.
  • Select an A2L file on the right side.
  • Click to map the model instance(s) to an A2L file.

To remove a model instance(s) from an A2L file

  • Select the mapped model instance(s) in an A2L file.
  • Click to unmap the model instance(s) from an A2L file.

To show unmapped model instances

  • Check the checkbox .
  • Only the model instance(s) which are not mapped to any of the A2L file is(are) displayed.

To delete a session or an A2L file

  • Click on the respective row.
  • A session cannot be deleted if it is mapped to an A2L file.

To edit an A2L file

The added model instance(s) to a system is displayed in the "Model Instances" panel of the A2L section.

  • Click .
  • The "A2L File" dialog box is displayed.
  • The name of an A2L file or session can be updated.
  • Click Update A2L File.

When you map a model instance(s) or edit the A2L file, the status changes to "Modified".

To generate the A2L file

  • Click Build on the Project menu.
  • After the successful execution, the A2L file is generated.

When you build a system, the status changes to "Generated" after the successful execution.

The "Generated" status will be changed to "Modified" state in the following cases:

  • When you switch from one target to another.
  • When a task is added/deleted/updated in the "Deployment" view and the period of a task is changed. In turn, the fastest task is updated in the XCP connection.
  • When the values of the existing model instance(s) is(are) updated.

In such cases, it is necessary to build a system again to generate it.

Refer to Generating an A2L File for limitations.

To export A2L file(s)

  • Select the A2L file(s) and click . All the available A2L files in a system are exported, if file(s) is(are) not selected.
  • The "Export A2L File" dialog box is displayed.
  • Select the location to export the file or click Browse to select it.
  • Click Export.
  • The "Replace or skip A2L File(s)" dialog box is displayed if the file(s) is(are) already available.
  • Click either "Replace the A2L File(s) in the destination" to replace the existing file(s) or "Skip these A2L File(s)" to ignore the file(s).
  • The generated A2L files are exported.

You can export only the generated A2L files.
