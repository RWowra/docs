Glossary

  • API
    Application Programming Interface
  • AR
    Acronym for AUTOSAR
  • ARXML
    File extension for AUTOSAR XML-formatted file(s)
  • ARXML files
    AUTOSAR XML files
  • AUTOSAR
    AUTOSAR is an evolving standard, which defines modular software architecture for automotive electronic control units. It is designed to increase the ease of software re-use and the deployment on different hardware. AUTOSAR increases the ease of software integration, but in a way mandates the use of tools to effectively handle its complexity. (See http://www.autosar.org )
  • CAN
    Controller Area Network (ISO 15765), a multi-master broadcast serial bus standard for connecting ECUs.
  • COSYM
    Co-simulation of Systems
  • DLL
    Dynamic Link Library
  • FMI
    FMI is a Functional Mockup Interface (FMI) that provides a standardised interface to be used in computer simulations to develop complex mechatronic systems.
  • FMU
    Model is converted to ".fmu" file with the help of importer. This ".fmu" file is called as Functional Mockup Unit (FMU).
  • GUI
    Graphical User Interface
  • GUID
    GUID is a Globally Unique Identifier. It is a 128-bit integer number used to identify a model.
  • HiL
    Hardware in Loop: HiL refers to a simulation of a closed loop system where one part of the system is in the actual hardware/ECU while the plant is a model in the computer.
  • ICE
    Internal Combustion Engine
  • LMD
    LABCAR model Description
  • Mechatronic
    Mechatronics is a design process that includes a combination of mechanical engineering, electrical engineering, control engineering and computer engineering. Mechatronics is a multidisciplinary field of engineering, which means that it rejects splitting engineering into separate disciplines.
  • MiL
    Model in Loop: Simulation of a system where all executable blocks are in the form of models such as simulink blocks etc.
  • Model
    Model is a basic unit of the system which represents a domain specific component in a product.
  • Native Model Or Model
    Native Model or Model is a basic unit of the system which represents a domain specific component. It is a graphical representation of a mathematical equation in the form of a block diagram.
  • NIC
    Network Integration CAN
  • NM XML
    Network Module XML file
  • Offline Simulation
    Imitation of real world processes or systems through mathematical models in a computer where time update/increment is in complete control of the simulator process is referred to as offline simulation.
  • RBS
    Restbus Simulation
  • Real-time Simulator
    A model runs at the same rate as the actual physical system in the Real-time Simulator.
  • RT Simulation
    Simulation of a system where the time update/increment is controlled by the OS of the computer such that it matches real-world time is referred to as real-time simulation.
  • SiL
    Software in Loop: Simulation of a system where the sub-systems are in the form of software files which subsequently can be programmed on actual ECUs.
  • Simulation
    Simulation is the imitation of operation of real-world processes or systems over time in a computer using mathematical/logic/table based models.
  • SIMULINK
    SIMULINK is an environment for multi-domain simulation and model-based design. It supports simulation, automatic code generation, and continuous test and verification of embedded systems.
  • Solver
    Solver is a generic term indicating a piece of mathematical software, possibly in the form of a stand-alone computer program or as a software library that 'solves' a mathematical problem
  • SUT
    System Under Test
  • Target Node
    Simulation Target Node is the address of the server/machine on which the simulation is being executed. In the present version, it is the local machine or the local host.
  • Tunable Parameter
    A Tunable Parameter is constant between external events (ModelExchange) and Communication Points (Co-simulation) due to changing variables. In other words, value of tunable parameters can be changed during simulation at runtime due to external events.
  • UDP
    User Datagram Protocol
  • VECU
    The ETAS Virtual ECU is an open and expandable Eclipse-based tool environment, enabling integration, configuration, and execution of the ECU software on the PC. The key function of this solution is the generation of an ETAS Virtual ECU software and configuration, with the combination of actual ECU application software. This permits a PC-based behavioral simulation of a complete ECU within a Windows and Linux environment.
  • XiL
    XiL refers to a kind of simulation where models can be in the form of software or hardware or plant models. This is a combination of MiL, SiL and HiL simulation types.
  • XML
    eXtended Markup Language
